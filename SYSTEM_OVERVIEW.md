- **Streamlit**: Python 기반 웹 프레임워크

- **OpenAI GPT-4**: 뉴스 분석 및 보고서 생성

- **Telegram Bot API**: 자동 알림 발송

### 3. **데이터 관리**
- **CSV 기반 DB**: 뉴스 히스토리 관리 (최근 200건)
- **JSON 마스터 데이터**: 언론사/담당자 정보 관리
- **자동 백업**: 과거 대응 이력 누적 저장

### 4. **배포 환경**
- **Streamlit Cloud**: 무료 클라우드 호스팅
- **GitHub**: 코드 버전 관리 및 자동 배포
- **환경변수 암호화**: API 키 안전 관리

## 🚀 5대 주요 기능

### 1️⃣ **뉴스 모니터링** (자동화 핵심)
**기능**: 포스코 관련 뉴스를 3분마다 자동 수집 및 분석

**프로세스**:
```
1. Naver API 호출 (5개 키워드 동시 검색)
   └─ "포스코인터내셔널", "포스코홀딩스", "포스코", "포항제철", "POSCO"

2. 신규 기사 감지 (URL 기반 중복 제거)
   └─ 최근 6시간 이내 기사만 선별

3. 텔레그램 알림 발송
   └─ [매체명] 기사제목
   └─ 발행시간 + 링크

4. AI 기사 요약 생성
   └─ 주요 내용 3줄 요약
   └─ 긍정/부정/중립 판단
```

**수집 데이터**:
- 발행일시, 매체명, 검색키워드, 기사제목, AI 요약, URL
- **최대 100건** 수집 → **상위 200건** DB 저장

**자동화**:
- ✅ 180초(3분) 자동 새로고침
- ✅ 신규 기사만 텔레그램 알림
- ✅ 중복 알림 방지 (URL 기반)

---

### 2️⃣ **이슈발생보고 생성** (AI 핵심)
**기능**: 언론사·기자·이슈를 입력하면 8단계 위기대응 보고서 자동 생성

**입력 정보**:
- 언론사명 (예: 조선일보)
- 기자명 (선택)
- 이슈 내용 (예: "전기차 배터리 화재 관련 보도")

**AI 생성 보고서 구조** (8단계):
```
1. 상황 요약
   └─ 핵심 이슈를 한 문단으로 정리

2. 언론사 정보
   └─ 매체 성향, 영향력, 주요 독자층 분석

3. 사실관계 확인
   └─ 확인된 사실 vs 미확인 정보 구분

4. 쟁점 분석
   └─ 핵심 쟁점 3가지 도출

5. 대응 방향
   └─ 즉시 대응 / 모니터링 / 적극 해명 중 선택

6. 메시지 초안
   └─ 대외 발표용 메시지 3가지 안 제시

7. 후속 조치
   └─ 단기/중기/장기 대응 계획

8. 담당 부서
   └─ 내부 데이터 기반 담당 부서 및 연락처 매칭
```

**처리 시간**: 약 30초 ~ 1분

**활용 시나리오**:
- 긴급 보도 대응: 기자 전화 받고 즉시 보고서 생성
- 사전 대응: 예상 이슈에 대한 시뮬레이션 보고서 작성
- 교육 자료: 신입 직원 대응 훈련용

---

### 3️⃣ **언론사 정보 검색**
**기능**: 언론사명 입력 → 담당 기자 리스트 및 연락처 자동 조회

**검색 가능 정보**:
- 언론사명
- 담당 기자 이름
- 전화번호
- 이메일
- 팩스번호

**데이터 출처**: `master_data.json` (200개 이상 매체 등록)

**검색 결과**:
```
📰 조선일보
├─ 김철수 기자
│  ├─ 📞 02-724-XXXX
│  └─ ✉️ chulsoo@chosun.com
└─ 이영희 기자
   ├─ 📞 02-724-YYYY
   └─ ✉️ younghee@chosun.com
```

**편의 기능**:
- 전화번호 원클릭 복사
- 이메일 자동 메일러 연동
- 담당 기자 즐겨찾기

---

### 4️⃣ **담당자 정보 검색**
**기능**: 이슈 키워드 입력 → 내부 담당 부서 및 담당자 자동 매칭

**검색 방식**:
```
입력: "배당"
  ↓
AI 분석: "배당" = 주주 관련 이슈
  ↓
매칭: IR그룹 유근석 담당자
  ↓
출력:
  부서: IR그룹
  담당자: 유근석
  전화: 02-3457-8114
  이메일: gs.yu@posco-inc.com
```

**등록된 부서 예시**:
- IR그룹: 배당, 주가, 투자자 관계
- 전략기획그룹: 경영 전략, M&A
- 홍보실: 언론 대응, 보도자료
- ESG그룹: 환경, 사회공헌, 지배구조
- 법무그룹: 소송, 법률 자문

**활용**:
- 긴급 이슈 발생 시 즉시 담당자 연락
- 크로스체크: 여러 부서 협업 필요 시 일괄 조회

---

### 5️⃣ **기존 대응이력 검색**
**기능**: 과거 유사 사례 검색 → 대응 방법 참고

**검색 데이터**: `언론대응내역.csv` (누적 사례)

**검색 조건**:
- 발생 유형 (예: 환경 이슈, 노사 갈등, 안전사고)
- 대응 단계 (예: 초기 대응, 확대 대응, 종결)
- 현업 부서

**검색 결과 예시**:
```
순번: 42
발생일시: 2024-11-15
발생유형: 환경 이슈
현업부서: ESG그룹
이슈내용: 제철소 대기오염 관련 시민단체 성명
대응결과: 환경개선 계획 발표 후 종결
```

**학습 효과**:
- 과거 성공 사례 벤치마킹
- 실패 사례 재발 방지
- 대응 시간 단축

### **시나리오 1: 긴급 보도 대응**
```
1. 텔레그램 알림 수신
   └─ "[조선일보] 포스코 화재사고 발생"

3. 이슈발생보고 생성 메뉴
   ├─ 언론사: 조선일보
   ├─ 기자명: (텔레그램에서 확인)
   └─ 이슈: "화재사고 관련 보도"

4. AI 보고서 생성 (30초)
   └─ 8단계 보고서 자동 작성

5. 담당자 정보 검색
   └─ 안전관리팀 담당자 즉시 연락

6. 기존 대응이력 검색
   └─ 과거 유사 사례 대응 방법 참고

```

```
1. 매일 오전 9시 웹 접속

   └─ 어제부터 오늘까지 신규 기사 확인
```
