## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 시스템명 | P-IRIS (포스코인터내셔널 언론대응 인텔리전스 시스템) |
| 유형 | AI 기반 언론 모니터링 · 위기관리 웹 애플리케이션 |
| 배포 환경 | Streamlit Cloud (`poscointlwh.streamlit.app`) |
| 핵심 기술 | GPT-4.1 · 네이버 API · GitHub Actions 자동화 |

포스코인터내셔널 언론홍보 담당자가 **뉴스 수집 → 이슈 분석 → 보고서 생성**까지 하나의 웹 플랫폼에서 처리할 수 있도록 구축한 원스톱 언론대응 인텔리전스 시스템이다.

## 2. 배경 및 문제 정의

### 2.1 기존 업무 방식의 한계

| 문제 영역 | 상세 내용 |
|----------|-----------|
| **수동 모니터링** | 담당자가 매일 직접 포털·검색엔진을 검색해 기사를 수집. 수십 개 키워드를 반복 검색하는 데 하루 수 시간 소요 |
| **대응 골든타임 손실** | 자동화된 감지 체계 부재로 이슈 발생 직후 신속 대응이 어려움. 기사가 확산된 뒤 인지하는 경우 발생 |
| **정보 파편화** | 기자 연락처·과거 대응이력·부서 담당자 정보가 개인 파일·메모 등 각자 관리되어 공유·검색 불가 |
| **보고서 수기 작성** | 이슈 발생 시 임원 보고용 문서를 매번 수기로 작성. 형식 불일치 및 작성 지연 반복 |

### 2.2 핵심 질문

> "언론홍보 담당자가 이슈를 **얼마나 빠르게 인지**하고, **얼마나 정확하게 대응**하며, **얼마나 효율적으로 공유**할 수 있는가?"

## 3. 목표

### 3.1 비즈니스 목표

1. **대응 골든타임 확보** — 뉴스 수집 주기를 3분 이내로 단축해 이슈 초기 대응력 강화
2. **업무 자동화** — 수동 뉴스 검색·보고서 작성 업무를 AI로 대체해 담당자 가용 시간 확보
3. **정보 일원화** — 기자 연락처·대응이력·부서 담당자를 단일 플랫폼에 집중 관리
4. **판단 품질 향상** — AI 기반 위기 등급·리스크·액션 제안으로 의사결정 지원

### 3.2 성공 지표 (Success Metrics)

| 지표 | 목표 |
|------|------|
| 뉴스 수집 주기 | ≤ 3분 (자동화 이전 대비 수 시간 → 3분) |
| 이슈 보고서 생성 시간 | 수기 작성 30분 이상 → AI 생성 1분 이내 |
| 키워드 인사이트 API 호출 비용 | 검색 1회당 GPT 1회 이하 유지 |
| 시스템 가동률 | Streamlit Cloud 월간 99% 이상 |

## 4. 사용자 및 이해관계자

### 4.1 사용자 유형

| 역할 | 설명 | 주요 사용 기능 |
|------|------|--------------|
| **PR 담당자** (주 사용자) | 언론홍보팀 구성원. 일상적인 뉴스 모니터링·대응·보고서 작성 | 전 기능 |
| **유관 부서 담당자** | IR팀·법무팀 등 언론 이슈와 연관된 사내 유관 부서 | 뉴스 현황 조회, 키워드 인사이트 |
| **임원·경영진** | 이슈 발생 시 요약 보고서 수신 | (보고서 수신 · Telegram 알림 수신) |

### 4.2 이해관계자

- **언론홍보팀 팀장** — 시스템 도입 결정, 운영 승인
- **IT/보안 담당** — API 키 보안, Streamlit Cloud 계정 관리
- **OpenAI / Naver / Telegram** — 외부 API 제공사

## 5. 기능 범위

### 5.1 In-Scope (구현 대상)

| # | 기능 | 설명 |
|---|------|------|
| 1 | **뉴스 모니터링** | 포스코인터내셔널 관련 키워드 자동 수집·분류, 텔레그램 실시간 알림 |
| 2 | **키워드 인사이트** | 임의 키워드 입력 시 AI 브리핑 자동 생성 (API 최소 호출) |
| 3 | **이슈보고 생성** | 이슈 정보 입력 → AI 임원 보고용 문서 자동 작성 |
| 4 | **언론사 정보** | 주요 출입기자 및 언론사별 연락처 검색 |
| 5 | **담당자 정보** | 내부 부서별 PR 담당자 정보 조회 |
| 6 | **대응이력 검색** | 과거 언론대응 사례 키워드·단계별 검색 |

### 5.2 Out-of-Scope (현재 미구현)

- SNS(트위터·유튜브 등) 모니터링
- 외부 언론사 시스템 직접 연동 (ERP, CMS)
- 다국어(영문) 뉴스 수집
- 모바일 전용 앱 (현재 웹 반응형으로 제공)

## 7. 기대 효과

### 정량적 효과

| 항목 | 개선 전 | 개선 후 |
|------|--------|--------|
| 뉴스 모니터링 시간 | 하루 수 시간 (수동 검색) | 자동화 (3분 주기) |
| 이슈 보고서 작성 | 30분~1시간 (수기) | 1분 이내 (AI 자동 생성) |
| 기자/담당자 정보 조회 | 개인 파일 탐색 (수 분) | 플랫폼 내 즉시 검색 |
| 과거 대응사례 참고 | 파일 탐색·구두 확인 | 키워드 검색 즉시 조회 |

### 정성적 효과

- 이슈 초기 대응력 강화 → 언론 리스크 통제력 향상
- 팀 내 정보 공유 문화 정착 → 담당자 의존도 분산
- AI 기반 위기 등급 제공 → 경영진 신뢰 의사결정 지원
